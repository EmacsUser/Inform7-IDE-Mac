22.6. Understanding

Understand ... when ... does not allow callings as yet, so the following fails and perhaps should work:

Understand the tone property as describing a letter (called the subject) when we have examined the subject.

Disambiguation is currently problematic, in the sense that there is no equivalent for some of the control provided in I6 by ChooseObjects. It would just be nice to be able to convey to the parser which objects to prefer under what circumstances: prefer edible things when eating, etc. Can at present add

Understand "eat [edible thing]" as eating.

which does force preference of edible objects; but this breaks the "held" requirement. Besides, it'd be neat to be able to make more complicated instructions about what to prefer.[ES]

Also, I would not mind, though I realize this may be heinous, if there were a disambiguation activity, so that I could write "Rule for printing the name of something while disambiguating..." or whatever. [ES]
