4.20. Recap of the action sequence

To recap, actions are normally generated when the player types a command, but they can also be started by the "try" phrase. What then happens is as follows:

1. If any before rule applies then it is obeyed.

2. If the action needs light to see by, that is checked; if it needs physical access to its objects, that too is checked.

3. If any instead rule applies then it is obeyed and the action is automatically stopped.

4. The "check" rules for the given action are obeyed, to see if it is possible. For example, the "check taking" rules will prevent the player from taking something which is fixed in place.

5. The "carry out" rules for the given action are obeyed. For example, the "carry out taking" rules actually move the given object into the player's possession. The action is now declared a success.

6. If any after rule applies then it is obeyed and the action is automatically stopped.

7. The "report" rules for the given action are obeyed. For example, the "report taking" rules always just say "Taken."

In practice we can forget about the check, carry out and report rules unless we are creating new actions or meddling with the definition of existing ones: almost everything we need to do can be done with before, instead and after rules.
