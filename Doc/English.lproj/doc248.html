17.9. Fuller documentation in the Contents index

The Contents index contains a brief list of all extensions used, their authors and version numbers. For instance:

Standard Rules version 1/040731
Locksmith by Emily Short
Tupperware by Emily Short

However, fuller documentation is also given if the extensions in question provide it. This text should be written as briefly as possible, while giving examples wherever appropriate. Stylistically, it should ideally follow the model of the main Inform documentation: just as an extension expands the standard rules, so its documentation expands this manual. "We need..." is preferred to "You need...", and so on: we're all in this together.

The text should appear at the foot of the extension file, after the compulsory end sentence. The first paragraph should consist of the single word "<Documentation>". For instance, the "Ducking Action" example might end:

The Ducking Action ends here.

<Documentation>

This is a modest extension, with much to be modest about. It allows us to use a new action for ducking, as in ducking the player's head (not as in ducking a witch). Ducking will do nothing unless rules are added:

+   Instead of ducking in the Shooting Gallery, say "Too late!"

We obtain indented code examples (in blue) by beginning a line with a plus sign followed by a tab (not by spaces). A double indentation can be got with plus, tab, plus, tab, and so forth.

+   Before opening or pushing the chest when something is on the lid:
+   +   say "(first taking [the list of things on the lid])[line break]";
+   +   repeat with item running through things on the lid begin;
+   +   +   try taking the item;
+   +   end repeat.

will be rendered thus:

Before opening or pushing the chest when something is on the lid:
    say "(first taking [the list of things on the lid])[line break]";
    repeat with item running through things on the lid begin;
        try taking the item;
    end repeat.
