17.19. The return context

I7 has a number of conceptually different ways to define code - as phrases which are not compiled inline, as phrases to decide conditions, as rules and so on. All these are compiled to I6 routines, and return values have different meanings in the different cases. For type-checking reasons, I7 needs to detect when a return might occur in order to make sure that a condition does not end with the wrong kind of result, and so on.

Inline phrases which cause a return should declare what happens like so:

To stop the action: (- rtrue; -) - outcome.

"Outcome", used for rules, is one of four possibilities. The others are "void" (a return with no value), "value" (with a value) or "condition" (used to signal a result of a phrase to decide whether something is true). Thus the phrases "yes" and "stop the action" have the same definition, but a different context:

To yes: (- rtrue; -) - condition.
