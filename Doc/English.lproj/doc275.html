18.9. Cover art

Accompanying files are not the only things which can be included in a "release along with" sentence:

Release along with cover art, a file of "Collegio magazine" called "Collegio.pdf" and a file of "The mating call of the green wyvern" called "Mating Wyverns.mp3".

Cover art for a work should be prepared in either JPEG (".jpg") or PNG (".png") format, and we recommend that it should be square, like a music album cover. Programs which notice the cover art for a work of IF are likely to scale this up or down as convenient for their own display purposes, but it would be helpful to provide the original art at 960 by 960 resolution. (Remember, it is intended for use on screen, so providing art at magazine full-page print quality will add nothing and will only make the resulting release an annoyingly large download: even as it is, 960x960 will print out as a pretty respectable CD insert sleeve.)

If we are providing cover art, or asking Inform to construct a website, then we need to collate the materials in a suitable folder. Suppose that our project is called "Magician.inform", a project file which happens to be placed in the folder "Works in Progress". Then we should set up the following arrangement:

Works in Progress
    Magician Materials
        Collegio.pdf
        Cover.jpg
        Mating Wyverns.mp3
        Small Cover.jpg
    Magician.inform

(supposing that, as in the previous examples, "Collegio.pdf" and "Mating Wyverns.mp3" are the filenames of two feelies that accompany the release). "Small Cover.jpg" should be a reduction of the main cover art image "Cover.jpg" to exactly 120 by 120 pixels.

At present there's no support for PNG format covers, but that will be added.
