12.12. Beginning and ending activities manually

There are times when it is not convenient to write a suitable "for ..." rule, or when we cannot be bothered. For such times we are allowed to write:

begin the analysing activity with the dustbin;
...
end the assaying activity with the dustbin;

but we need to be careful to follow the golden rules: all activities must end, they must never last longer than a turn, and if activity B starts during activity A then it must also finish during activity A. We must also be careful to make sure that if an activity applies to something, then it begins and ends with the same something (the dustbin, in the above example).

The before rules are considered at the time of the "begin ..." phrase; the after rules at the "end ...". What, then, of the "for" rules? In the above setup, they would simply be ignored. We can make them effectual like so:

begin the analysing activity with the dustbin;
...
if the analysing activity with the dustbin is required begin;
    ...
end if;
...
end the assaying activity with the dustbin;

The "for" rules are invoked when the test about requirement is carried out: the condition is true only if no rule has intervened. Thus the "..." inside the "if" is the place to put the default behaviour of the activity.
