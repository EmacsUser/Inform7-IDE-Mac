21.3. Sam Ashwell - issues with order of declarations

Reported by: Sam Ashwell / Date: 25 November 2005

Nature: The source looks legal, but generates Problem messages

Build: 2V60

Platform: Mac OS X

Code example:

"Test" by Sam Kabo Ashwell

Testroom is a room.

Eating a mustelid is revolting behaviour.
Mustelid is a kind of thing.

What goes wrong:

Problem. You wrote 'Eating a mustelid is revolting behaviour' : but

this seems to say that two different things are the same, like saying

that 'the hawk is the handsaw'.

Comments:

If the two statements are reversed, like so:

Mustelid is a kind of thing.

Eating a mustelid is revolting behaviour.

then the game will compile as expected.

It seems to me that a requirement to declare things (apart from a few

things like the title) in a particular order is a pretty hefty

weakness. If this weakness is too problematic to overcome, however, I

think it only fair to prominently mention it in the docs. If it's

going to be necessary to declare all my kinds at the start of the

source, I'd like to know about it before I start organising my code.

SKA
