22.5. Convenient but non-fundamental semantics

"There", as in: if there is an open door in the Chamber, ... This exists already for table cells, so it seems unnatural that it doesn't for general predicates.

Paragraph sorting in room descriptions.

This is conceptually, even if not in execution, related to the name-printing issues, and to my idea about having the list writer allow for some internal ordering of the list. Again we are trying to allow the author to do some structuring of how the output text reflects the world model, by indicating (among other things) what is important, what is of ordinary relevance, and what should be suppressed. An object can be important and therefore moved for reasons of art to the first or last place in the room description; a property or relation can be important and therefore mentioned in the object name; an object or property or relation can be as yet unknown, and therefore kept secret until the player has done the right things to uncover it.[ES]

Are we now happy with the "pair of gloves" problem, Richard Tucker's yardstick for the adequacy of an IF design system?
