7.15. Arithmetic on new kinds of value

At the start of this chapter we saw that "+" could be used to combine two number values into one:

change the score to the score + 4

In Inform, "+" is a phrase like any other. Its definition behind the scenes begins:

To decide which number is (X - number) + (Y - number): ...

Thus Inform will only recognise "+" as valid if it applies to two numbers, which is as it should be. We cannot normally apply "+" to values like these:

Colour is a kind of value. The colours are red, orange, yellow, green, blue, indigo and violet.

However, we do automatically have two ways to make new colours from old:

the colour after C
the colour before C

produce the next, and the previous colour respectively, in order of their declaration. Thus "the colour after red" is orange; "the colour before blue" is green. The values wrap around, that is, "the colour before red" is violet and "the colour after violet" is red.

We are also allowed to use number-like comparisons with new kinds of value. Thus "red < yellow" is true, while "green >= violet" is not.
