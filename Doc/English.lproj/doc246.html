17.11. New activities

The original Inform 6 library allowed itself to be customised by providing "entry point routines"; in Inform 7, this method was replaced by activities. If an extension contains any elaborate mechanisms then it, too, should consider setting them up as activities. This is easily done. For instance:

Assaying is an activity. The last for assaying rule: ...

Analysing something is an activity. The last for analysing rule: ...

The former activity does not apply to any particular object, while the latter does: the difference is in the word "something". (Equivalently, it could end "of something", as the standard rules' activity "listing contents of something" does.) We can then carry out the activities with phrases like so:

carry out the assaying activity;
carry out the analysing activity with the dustbin;

It may be worth noting that an activity can still be worth creating even when it is hard to imagine useful rules attached to it, because the activity also gives us a new context for other rules. The designer might well be grateful for the chance to define a "rule for printing the name of a container while assaying", for instance.
