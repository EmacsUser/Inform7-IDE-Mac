11.13. Defining loops

The two I7 versions of "while" are defined in the Standard Rules as follows:

To while (c - condition) repeatedly (ph - phrase): (- while {c} {ph} -).

To while (c - condition) begin *: (- while {c} -).

The second case is the one which involves a block of code, to be ended later with an "end while". The asterisk signals to I7 that such a block is expected. The I6 code should be such that an I6 code block, starting and ending with a brace, can legally follow it.

Variables can be created as part of a loop which exist only within it. All that needs to be done is to supply their type and declare them to be "variable":

To repeat with (loopvar - number variable) running from (v - number) to (w - number) begin *: (- for ({loopvar}={v}: {loopvar}<={w}: {loopvar}++) -).

To repeat with (loopvar - object variable) running through contents of (v - object) begin *: (- objectloop ({loopvar} in {v}) -).

I7 will compile this using a local variable for "loopvar": it may well re-use this variable later for other loops, so its value cannot be relied upon outside of the code block following the loop.
