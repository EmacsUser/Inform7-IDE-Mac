6.8. Giving instructions to other people

In interactive fiction, it is conventional to ask other characters to do something like so:

> marcel, put the biscuit in the tea
> will, go west

and so on. Clearly "will, go west" should not produce the same action as "go west", because a different person will be trying it: this person is called the "actor", and while the actor is ordinarily the player, here it is the character of Will. The actions produced are:

> go west -> going west
> will, go west -> asking will to try going west

We can thus write rules like so:

Instead of asking will to try going west, say "Will scratches his head, baffled."

which apply only to the command "will, go west" and do not affect the command "go west". Actions carried out by other people get as far as completing the "instead" rules but then, if no rule has intervened by then, fail like so:

> will, go west
Will has better things to do.

Similarly, we can also have phrases like so:

try asking the Sybil to try waiting

which simulate the player having typed "sybil, wait".
