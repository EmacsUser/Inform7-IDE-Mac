15.11. Reaching inside rules

So far we have seen rules which apply to actions, and rules which do not apply to anything specific - "every turn" rules, for instance. We can also have rules which apply to a specific object.

An example of rules applying to given objects is provided by the way that Inform decides whether the player can reach something or not. For instance, suppose the following:

The Laboratory is a room. In the Laboratory is a conical flask. The flask is closed and transparent. In the flask is an antibumping granule.

The player will be able to examine the granule but not to take it, as that would require reaching through glass. Internally, what happens if the player tries to take the granule is this: a rulebook called the "reaching inside rules" is applied to the conical flask. This normally contains the following three rules:

the can't reach inside closed containers rule
the can't take people's possessions rule
the can't take component parts rule

Two of these rules apply only to actions of taking and removing: if the player only wants to push the granule, for instance, only the "can't reach inside closed containers" rule is applicable. This is the gist of it:

The can't reach inside closed containers rule: if the object in question is a closed container begin; say "[The object in question] is closed!"; deny access; end if.

In such a rule the "object in question" is the object to which the rule is being applied. Note that a reaching inside rule can "deny access" (stopping with fail), or "allow access" (stopping with success), or neither, in which case the decision is left up to any subsequent rules in the rulebook to make. If none of them decide, access is allowed.
