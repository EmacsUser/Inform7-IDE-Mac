7.16. Making new kinds of arithmetic

Because "+" is a phrase like any other, we are free to extend its definition to other kinds of value beyond numbers (which might be called "overloading the operator +" in computing jargon). The following example game defines a way to add a colour to a number, producing another colour:

"Overloading"

Colour is a kind of value. The colours are red, orange, yellow, green, blue, indigo and violet.

To decide which colour is (C - a colour) + (N - a number):
while N is not 0 begin;
if N < 0 begin; let C be the colour before C; let N be N + 1; end if;
otherwise; let C be the colour after C; let N be N - 1; end if;
end while;
result C.

The Counting House is a room. "Prominent on the wall is a sign reading: The Magic Number is [the magic number]."

The magic number is a number that varies.

In the House is a lever. Instead of pushing the lever: change the magic number to the magic number + 1; say "The magic number increases to [magic number]." Instead of pulling the lever: change the magic number to the magic number - 1; say "The magic number decreases to [magic number]."

The beam colour is a colour that varies.

The prism is in the House. "A prismatic wedge of glass (oh, all right, a prism) splits a beam of white light so that a fine line of [beam colour] light falls across the sign."

The button is in the House. Instead of pushing the button: change the beam colour to the beam colour + the magic number; say "The prism swivels, and after a blur the beam falling on the sign settles at [beam colour]."

The prism, button and lever are fixed in place.
